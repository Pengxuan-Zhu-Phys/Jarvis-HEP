# Jarvis-HEP v1.6.5 Remaining P1 Breakdown

Last updated: 2026-03-03  
Scope: close out remaining v1.6.5 sampler hardening P1 items without redesign.

## P1-1. Reduce Nested Executors And Over-Subscription Risk

Status: `DONE`

### Current State

- Dynesty/MultiNest path already adapted to `Factory`-aligned async submission.
- Remaining nested execution path still exists in module runtime:
  - `Factory` worker -> `ModuleManager.execute_workflow(...)`
  - `ModulePool.execute(...)` -> internal `ThreadPoolExecutor.submit(...).result()`
- This keeps an extra thread-pool layer in hot execution path and may amplify scheduling overhead under short tasks.

### Task Decomposition

1. Refactor `ModulePool` responsibility split:
   - Keep: instance install/reload/rent/return.
   - Remove from hot path: inner `executor.submit(instance.execute, ...).result()`.
2. Keep per-module instance safety:
   - enforce one-task-per-instance busy flag discipline.
   - avoid holding coarse locks during `instance.execute(...)`.
3. Preserve barrier semantics in `ModuleManager`:
   - same-layer modules still complete before next-layer execution and `nuisance_check`.
4. Add regression tests:
   - no deadlock under pool saturation.
   - module A saturation does not block module B execution path.
   - fail-fast behavior remains unchanged.

### Acceptance Criteria

- No inner `ThreadPoolExecutor.submit(...).result()` in module hot execution path.
- Runtime behavior compatible with existing sampler/Factory interfaces.
- Tests prove no cross-module blocking and no deadlock.

### Closure Notes

- Implemented in `jarvishep/modulePool.py` by executing `instance.execute(...)` directly on current Factory worker thread.
- Kept asynchronous install path via background executor and preserved instance rent/return safety.
- Regression guard added in `tests/test_v165_remaining_p1.py`.

## P1-4. Harden Grid Boundary Behavior For 0/1 Endpoints

Status: `DONE`

### Current State

- Grid sampler currently produces exact endpoints from `np.linspace(0., 1., steps)`.
- Exact 0/1 may trigger infinities for transforms such as `logit`/`ppf` in variable mapping.

### Task Decomposition

1. Add endpoint guard in Grid path:
   - clamp unit-cube values to open interval `(eps, 1-eps)` before distribution mapping.
2. Keep deterministic spacing intent:
   - endpoint clamp should be minimal and numerically stable (`eps` documented).
3. Add focused tests:
   - transformed values are finite for edge bins.
   - no behavioral regressions for typical mid-range points.

### Acceptance Criteria

- Grid no longer emits non-finite mapped values due to exact 0/1 endpoints.
- Boundary handling strategy is documented in sampler tasklist and tests.

### Closure Notes

- Implemented in `jarvishep/Sampling/grid.py` by clamping grid unit-cube endpoints to open interval `(eps, 1-eps)` and centering single-step grids at `0.5`.
- Regression guard added in `tests/test_v165_remaining_p1.py` for finite Normal/Logit mappings.
