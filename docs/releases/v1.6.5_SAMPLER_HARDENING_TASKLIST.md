# Jarvis-HEP v1.6.5 Sampler Hardening Tasklist

Last updated: 2026-03-03
Owner: Jarvis-HEP maintainers

## Scope

Patch line focused on sampler correctness, robustness, and maintainable concurrency.

- In scope:
  - Fix sampler/core interface mismatches that can break runtime.
  - Eliminate known correctness bugs in sampler paths.
  - Add hardening tests for sampler contracts and execution safety.
  - Improve concurrency strategy where current behavior is fragile or wasteful.
- Out of scope:
  - New sampler algorithms.
  - Breaking YAML schema changes.
  - Visualization/UI-only work.

## P0 (Blockers)

1. `[DONE]` Unify `WorkerFactory.submit_task` call contract across all samplers and test helpers.
   Acceptance:
   - All call sites pass one `sample_info` payload with explicit `Index`/`Name` fields.
   - No remaining old two-argument call sites.
   - Existing random/grid/mcmc/tpmcmc/dnn/dynesty flows run without adapter hacks.

2. `[DONE]` Restore/define sampler selection API contract.
   Acceptance:
   - `super().evaluate_selection(...)` calls are either implemented in base classes or replaced by a valid shared utility.
   - Selection behavior is deterministic and tested in at least one sampler.

3. `[DONE]` Fix DNN sampler correctness regressions.
   Acceptance:
   - `check_output_in_likelihood` gating logic is correct (no inverted semantics).
   - Package import path in DNN code uses package-safe import (`jarvishep...`) and works in installed mode.
   - DNN path has a minimal smoke test.

4. `[DONE]` Enforce sampler resource lifecycle (`Sample.close()`).
   Acceptance:
   - Every sampler that creates `Sample(...)` closes it in normal and error paths.
   - No persistent logger/file-handle growth across repeated runs in one process.

5. `[DONE]` Tighten module execution failure policy in workflow path.
   Acceptance:
   - Module exceptions are not silently swallowed.
   - Failure policy is explicit (fail-fast by default, optional continue mode only if documented and configured).
   - Unit test verifies failure visibility.

## P1 (Hardening And Performance)

1. `[DONE]` Reduce nested executors and over-subscription risk.
   Acceptance:
   - Documented execution model for sampler pool vs module pool.
   - At least one layer of redundant thread-pool nesting removed or guarded.
   Breakdown:
   - See `docs/releases/v1.6.5_REMAINING_P1_BREAKDOWN.md` (`P1-1`).
   - Implemented target: moved `ModulePool` hot execution off inner `submit(...).result()` path while preserving instance rent/return semantics.

2. `[DONE]` Replace busy-wait loops with bounded blocking or event-driven checks.
   Acceptance:
   - No tight polling loops with very short sleep/timeout in hot sampler paths.
   - CPU idle overhead drops in no-work periods (validated by a simple benchmark/log timing).

3. `[DONE]` Fix Dynesty precision/mapping issues.
   Acceptance:
   - No unsafe `float16` truncation for posterior sample arrays.
   - `samples_u` and mapped outputs use correct source arrays.
   - Dynesty output schema remains backward-compatible.

4. `[DONE]` Harden boundary behavior in Grid sampler transforms.
   Acceptance:
   - Guard against `ppf/logit` infinities due to exact 0/1 endpoints.
   - Edge handling strategy is documented and tested.
   Breakdown:
   - See `docs/releases/v1.6.5_REMAINING_P1_BREAKDOWN.md` (`P1-4`).
   - Implemented target: clamped Grid unit-cube endpoints to open interval before variable transform mapping.

5. `[DONE]` Add sampler regression tests.
   Acceptance:
   - New tests cover interface contract, selection behavior, DNN gating, and one concurrency/failure path.
   - Test suite remains stable in local and CI runs.
